 SDLLibrary

Toka has a wrapper over the raw SDL library that allows for use of SDL drawing and events.

needs sdl

Primitive SDL functions:

  • SDL_Quit
    • -- n
  • SDL_Delay
    • value -- n
  • SDL_GetTicks
    • -- n
  • SDL_PollEvent
    • event -- n
  • SDL_WaitEvent
    • event -- n

Higher-level wrappers:

  • sdl_setup
    • height width --
    • Create a new surface and open the drawing window for it.
  • sdl_render
    • --
    • Render the scene on the drawing surface
  • sdl_surface_size
    • -- n
    • Return the size of the drawing surface
  • sdl_event_size
    • -- n
    • Return the size of the event structure. malloc this and assign a name to create an event structure for SDL_PollEvent or SDL_WaitEvent.
  • sdl_event_type
    • event -- typecode
    • Return a value corresponding to the type of event which happened.
  • putpixel
    • x y color --
    • Draw a single pixel on the screen.
  • circle
    • x y radius color --
    • Draw a hollow circle on the screen.
  • aline
    • x y angle distance color --
    • Draw a line starting at x,y and extending out distance at a specified angle.
  • hline
    • x y length color --
    • Draw a horizontal line starting at x,y
  • vline
    • x y height color --
    • Draw a vertical line starting at x,y
  • line
    • x1 y1 x2 y2 color --
    • Draw a line starting at x1,y1 and ending at x2,y2
  • box
    • x1 y2 x2 y2 color --
    • Draw a hollow rectangle.
  • get-mouse-coords
    • event -- x y
    • Get the x,y coordinates of the mouse (for mouse events)
  • get-keysym
    • event -- keycode
    • Get the value of the key that was pressed (for keyboard events)
  • rgb:
    • $hex -- value
    • Parse ahead and build a color code from the RRGGBB hex string that follows.
